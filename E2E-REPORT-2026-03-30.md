## 2. Найденные и Исправленные Баги

### BUG #1 — `eval --llm` judge всегда fail (CRITICAL → FIXED ✅)

**Корневая причина**: `resolveJudgeConfig()` в `eval-service.ts:48` возвращал невалидный model ID:
```
anthropic/claude-sonnet-4-5-20250929  ← НЕПРАВИЛЬНО (не существует на OpenRouter)
anthropic/claude-sonnet-4.5           ← ПРАВИЛЬНО
```

**Симптом**: Все 212 LLM-judged тестов получали `[ERROR] LLM unavailable` — probe call на line 133 падал с 400, empty catch block глотал ошибку.

**Исправление** (eval-service.ts):
1. Line 48: model ID `anthropic/claude-sonnet-4-5-20250929` → `anthropic/claude-sonnet-4.5`
2. Line 140: Добавлен `console.warn` с деталями ошибки вместо пустого catch

**Результат**: Eval --llm теперь работает: 82/100 (B), 168 passed, 42 failed.

### BUG #2 — L5 scan findings пустое поле Fix (MINOR, P3)

**Симптом**: `scan --llm` L5 findings показывают `Fix: ` без текста.
**Место**: Генерация finding для L5 документов — fix field не заполняется.

### BUG #3 — Doc quality не обновляется scaffold→draft (MINOR, P3)

**Симптом**: После ручного редактирования scaffold документов — passport по-прежнему показывает `doc_quality: "scaffold"`.
**Причина**: L2 checker требует значительного контента для upgrade.

## 4. Обзор Архитектуры

### 4.1 Сильные Стороны

- **Clean Architecture**: Порты/домен/сервисы/инфра — чёткое разделение
- **Evidence chain**: Криптографическая цепочка (ed25519 + SHA-256) — надёжно
- **Event bus**: Развязка компонентов через SSE
- **SDK middleware**: Типизированный pipeline хуков
- **Eval system**: 3 режима (det/llm/security) — хорошее покрытие

### 4.2 Критические Проблемы

| # | Проблема | Файл | Severity |
|---|----------|------|----------|
| 1 | Silent catch blocks — ошибки глотаются без логирования | eval-service:140, composition-root:108, chat-service:122 | HIGH |
| 2 | `as unknown as` кастинг без валидации | composition-root:514 (GuidedOnboardingState) | HIGH |
| 3 | `JSON.parse()` без Zod schema | passport-service:63, eval-service:204 | HIGH |
| 4 | `console.warn/error` вместо logger port | eval-service, llm-adapter | MEDIUM |
| 5 | Hardcoded model IDs в eval-service | eval-service:45-51 | MEDIUM |
| 6 | Нет тестов для judge fallback logic | eval-service:114-159 | HIGH |

2. **P0**: Добавить контекстное логирование в silent catch blocks (особенно eval judge)
3. **P1**: Написать тесты для judge fallback logic (3 уровня: dedicated → target → stub)
4. **P1**: Заменить `as unknown as` на `safeParse()` с валидацией
5. **P2**: Вынести hardcoded model IDs в config constants
