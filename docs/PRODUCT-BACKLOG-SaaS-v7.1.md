(3) Переключение вида: CTO видит одно, DPO — другое, разработчик — третье. Базовый дашборд уже работает на 75%. | 🔴 |

## 3. SPRINT 9 — РЕЕСТР + ВЫХОДНЫЕ ДОКУМЕНТЫ + РЕГУЛЯТОР

> **Цель:** Единый реестр AI систем, внешние compliance outputs (Badge, Vendor Requests), интеграция с регулятором. Пользователь может доказать compliance третьим сторонам.

| # | Фича | Что делаем | При |
|---|------|-----------|-----|
| **F39** | **Реестр AI систем** | Центральная страница: ВСЕ AI системы организации в одной таблице. Включает: системы, найденные CLI автоматически (Mode 1, из кода) + системы, добавленные вручную через wizard (Mode 3, SaaS-only вендорские tools вроде ChatGPT). По каждой системе: название, вендор, risk class, L-level (автономность 1-5), compliance score, Passport completeness %, lifecycle status (active/suspended/decommissioned), владелец. Кнопки: kill switch (приостановить систему), открыть Passport, начать FRIA. | 🔴 |
| **F46** | **Wizard регистрации AI системы (шаги 3-5)** | Продолжение существующего wizard'а (шаги 1-2 уже есть). Шаг 3: use case + какие данные обрабатывает + кто end users. Шаг 4: уровень автономности L1-L5 + ограничения + human oversight setup. Шаг 5: Review → сохранить. Каждый шаг показывает подсказку "Art.26(x) требует..." Всё сохраняется в Passport. | 🔴 |
| **F56** | **Расширенные поля Passport** | Добавить в схему Passport новые блоки данных: (1) `regulatoryContext` — страна, сектор, регулятор (MSA), номер регистрации в EU DB. (2) `incidents[]` — лог инцидентов. (3) `postMarketMonitoring` — есть ли план, частота, метрики. (4) `conformityAssessment` — тип, дата, документ. (5) `complianceRecords` — уведомление работников, AI-обучение, data assessment, human oversight. (6) `msaSubmissions[]` — что отправляли регулятору, когда, статус. Эти поля нужны для полноценного Audit Package. | 🔴 |
| **F47** | **Помощник регистрации в EU Database** | EU AI Act обязывает зарегистрировать high-risk AI в европейской базе (Art.49). Это ~40 полей. Наш помощник: (1) предзаполняет поля из Passport (60-90%). (2) Показывает чеклист "что готово / что не хватает". (3) Даёт ссылку на EU Database для финальной отправки. (4) После регистрации пользователь вносит номер → обновляется Passport. | 🟠 |
| **F50** | **Compliance Badge** | Embeddable бейдж "AI Act Compliant" для сайта компании. Уровень L1 (self-assessed, бесплатно для Starter): показывает что компания прошла самопроверку. Уровень L2 (verified, Enterprise): подтверждённый после аудита. Виджет: `<script src="complior.ai/badge/xxx">`. Публичная страница верификации: complior.ai/verify/xxx. Для B2B SaaS-компаний = ускоритель продаж ("наш AI compliant, вот бейдж"). | 🟠 |
| **F51** | **Запрос документации у вендора** | Deployer имеет право запросить у вендора AI инструмента документацию (Art.13, Art.26). Наш инструмент: (1) выбираешь AI tool из реестра, (2) система генерирует email-шаблон с юридическими ссылками на AI Act, (3) отправляешь прямо из SaaS или копируешь в свою почту, (4) трекинг статуса: отправлено → ожидание → получено. Полученная документация прикрепляется к Passport. Фидит Community Evidence (другие deployer'ы видят "вендор X предоставил документы"). Этой фичи нет ни у одного конкурента. | 🟠 |
| **F53** | **Справочник регуляторов** | Интерактивная база: в какой стране какой регулятор отвечает за AI Act. Пользователь указывает страну + сектор → система показывает: название MSA (Market Surveillance Authority), контакт (email, адрес, сайт), как отправлять отчёты (портал / email / почта), какие документы могут запросить, дополнительные национальные требования (напр. Италия: уведомить работников по Art.12 итальянского закона). 27 стран EU + 3 EEA. Автоматически привязывается к Passport (через поле country + sector). | 🟠 |
| **F57** | **Генератор QMS** | Wizard для создания документа "Система управления качеством" (Quality Management System). Art.17 AI Act, AESIA Guide #4. Пошагово: (1) информация об организации, (2) список AI систем (из реестра), (3) назначение ответственных за каждый процесс, (4) описание процессов: одобрение AI, управление изменениями, vendor management, обучение, внутренний аудит. Предзаполняется из данных организации и реестра. Результат: PDF-документ 20-40 стр. | 🟠 |
| **F58** | **Генератор плана управления рисками** | Для каждой AI системы: wizard создания Risk Management Plan. Art.9, AESIA Guide #5. Шаги: (1) выбор AI системы, (2) список рисков (предзаполняется из Scanner findings), (3) для каждого риска: вероятность, последствия, меры снижения, остаточный риск, (4) график пересмотра. Результат: PDF + structured data в Passport. | 🟠 |
| **F59** | **Генератор плана мониторинга** | Шаблон Post-Market Monitoring Plan. Art.72, AESIA Guide #13. Что мониторим (accuracy, fairness, errors). Как часто (weekly, monthly). Пороговые значения (ниже какого уровня — стоп). Процедура эскалации. Ответственный. Обратная связь от пользователей. Предзаполняется из Passport. Результат: документ 5-10 стр. | 🟠 |
| **F40** | **Дашборд готовности к сертификации** | Показывает: насколько организация готова к ISO 42001 и AIUC-1. Per-system: что сделано, что осталось. Управление evidence (загрузка доказательств). Реферальная ссылка на партнёров-сертификаторов. | 🔴 |
| **F63** | **Индикатор источника данных** | В SaaS дашборде рядом с каждым полем показывать откуда данные: "Источник: CLI scan, 27 фев" или "Источник: введено вручную". Помогает DPO понять какие данные автоматические (надёжные), а какие ручные (проверить). | 🟡 |
| **F06** | **Eva — AI-ассистент** | Чат-бот внутри платформы. Отвечает на вопросы по AI Act, помогает заполнить Passport, объясняет обязательства. Интерфейс чата уже есть (SVG-аватар, история чатов). Нужен backend: LLM интеграция, контекст из Passport и обязательств. | 🟠 |
| **F10** | **Eva Tool Calling** | Eva может вызывать функции платформы: "Eva, сгенерируй FRIA для HireVue" → Eva запускает генератор. "Eva, какой у нас score?" → Eva показывает данные из дашборда. | 🟠 |
| **F11** | **Онбординг и уведомления** | Wizard первого входа: "Добро пожаловать → Добавьте первую AI систему → Классифицируйте → Заполните Passport". Уведомления: "Новое обязательство", "Deadline через 30 дней", "Вендор ответил на запрос". UI уведомлений есть (3 new), нужна логика. | 🟠 |
