# ADR-005: Compliance Gate — ре-скан на каждое изменение

**Статус**: Принято
**Дата**: 2026-02-17
**Автор**: Marcus (CTO)

## Контекст

Существующие сканеры (Snyk, ESLint, CodeRabbit) проверяют код **после** написания — в CI/CD, в PR, по запросу. Ни один инструмент не проверяет комплаенс **во время** написания кода. Это создает разрыв: разработчик может написать сотни строк некомплаентного кода, прежде чем узнает о проблемах.

## Решение

**Compliance Gate** — каждое изменение кода автоматически запускает фоновый ре-скан. Score обновляется в реальном времени. Если score падает — TUI показывает предупреждение с точной статьей и предлагает авто-фикс.

### Поток

```
1. Пользователь (или LLM) изменяет файл через инструмент кода
2. Инструмент кода уведомляет Compliance Gate: "файл изменен"
3. Gate запускает ре-скан проекта (AST + правила, ~200мс)
4. Сканер возвращает: новый score, новые findings, diff от предыдущего скана
5. Gate обновляет дашборд: индикатор score, список findings
6. Если score упал — предупреждение с конкретной статьей + кнопка "Авто-фикс"
```

### Что запускает Gate

File Watcher (chokidar/inotify) отслеживает изменения файлов в проекте. Каждое изменение файла автоматически запускает Compliance Gate:
- Новый файл → ре-скан
- Изменение файла → ре-скан
- Удаление файла → ре-скан
- npm install (новый пакет) → ре-скан
- Git merge/cherry-pick → ре-скан

### Уникальность

| Инструмент | Проверяет | Когда |
|-----------|----------|-------|
| Snyk | Уязвимости | CI/CD, по запросу |
| ESLint | Качество кода | При сохранении, CI/CD |
| CodeRabbit | Качество PR | При создании PR |
| **Complior** | **Комплаенс с AI-регуляциями** | **На каждое изменение кода, в реальном времени** |

Ни один другой инструмент не делает проверку комплаенса в реальном времени во время написания кода.

## Обоснование

- **Мгновенная обратная связь**: разработчик видит влияние каждого изменения на комплаенс сразу, не через часы/дни.
- **Предотвращение, не обнаружение**: проблемы ловятся при создании, не при ревью.
- **200мс скан**: достаточно быстро для бесшовного UX. AST-парсинг + проверка правил — без LLM-вызовов в горячем пути.
- **Авто-фикс**: не просто предупреждение, а конкретное решение одним нажатием.
- **Детерминистичность**: один и тот же код → один и тот же score. Нет «AI думает» — только правила.

### Производительность

- Скан проекта: 50-200мс (33+ pattern rules + 45 banned packages + 5 cross-layer rules)
- HTTP-оверхед: ~5мс (localhost)
- Обновление TUI: <1мс (immediate mode рендеринг)
- Пользователь замечает задержку: нет (скан в фоне, UI не блокируется)

## Последствия

- **Позитивные**: уникальный дифференциатор, мгновенная обратная связь, превентивный подход, стимулирует adoption.
- **Негативные**: дополнительная нагрузка при каждом изменении, потенциальное замедление на огромных проектах.
- **Митигация**: скан инкрементальный (только измененные файлы), кэш AST, debounce при быстрых последовательных изменениях (100мс).
