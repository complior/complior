# ADR-003: Мульти-модельный роутинг

**Статус**: Partially Superseded — LLM chat removed in v8, but L5 scan routing and report generation routing remain relevant
**Дата**: 2026-02-17
**Автор**: Marcus (CTO)

## Контекст

Complior — coding assistant, который использует LLM для объяснений, генерации кода, написания отчетов. Разные задачи требуют разного соотношения качества и стоимости. Пользователь предоставляет свой API-ключ — стоимость для Complior: $0/сессию.

**КРИТИЧЕСКОЕ ПРАВИЛО**: LLM используется только для UX (объяснения, навигация, генерация документации, предложения). LLM НИКОГДА не принимает решения о комплаенсе. Все проверки детерминистичны (AST + правила). Это исключает юридические риски (прецедент DoNotPay FTC: $193K штраф).

## Решение

Мульти-модельный роутинг через Vercel AI SDK v6 с тремя уровнями:

### Уровень 1: Умные дефолты (P0)

Разные задачи → разные требования к точности/стоимости → оптимальная модель:

| Тип задачи | Оптимальная модель | Фолбэк | Токены |
|-----------|-------------------|--------|--------|
| Интерпретация результатов скана | GPT-4o-mini / Haiku | Любая дешевая | 500-1,000 |
| Объяснение регуляции | Sonnet / GPT-4o | Любая средняя | 1,000-2,000 |
| Генерация кода фикса | Sonnet / GPT-4o | Любая средняя | 500-1,500 |
| Нарратив отчета | Opus / o1 | Любая мощная | 2,000-4,000 |
| Простые Q&A | Haiku / Flash / mini | Любая дешевая | 200-500 |
| Диалог онбординга | Sonnet / GPT-4o | Любая средняя | 500-1,000 |
| Суммаризация сессии | Haiku / Flash / mini | Любая дешевая | 300-600 |

### Уровень 2: Отображение стоимости (P0)

После каждого взаимодействия: «Этот ответ использовал ~$0.02 (4K токенов). Итого за сессию: $0.08.»

Слеш-команда `/cost` — показывает разбивку стоимости сессии по типу задачи, использованной модели, общей сумме.

### Уровень 3: Мульти-модельный консенсус (P2)

Для критических сценариев комплаенса: одна и та же задача проходит через 2-3 модели, результаты сравниваются. Консенсус → высокая уверенность. Расхождение → флаг для ревью человеком. Только для сложных пограничных анализов, не для рутинного сканирования.

## Обоснование

- **Vercel AI SDK v6**: один импорт, все провайдеры (OpenAI, Anthropic, Google, Mistral, Ollama, OpenRouter). Стриминг + tool calling из коробки.
- **$0 стоимость для нас**: пользователь использует свой ключ. Мы просто маршрутизируем к оптимальной модели.
- **Прозрачность стоимости**: пользователь видит, сколько стоит каждое взаимодействие — доверие и контроль.
- **Умные дефолты**: простые вопросы → дешевая модель. Отчеты → мощная. Оптимизация без ручной настройки.

## Последствия

- **Позитивные**: оптимизация стоимости для пользователя, поддержка 6+ провайдеров, прозрачность расходов.
- **Негативные**: сложность маршрутизации, необходимость тестирования на разных моделях, разная точность.
- **Митигация**: дефолты покрывают 95% случаев, пользователь может переопределить модель, консенсус только для P2.
