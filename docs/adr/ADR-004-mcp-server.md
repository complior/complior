# ADR-004: MCP Server как канал дистрибуции

**Статус**: Принято
**Дата**: 2026-02-17
**Автор**: Marcus (CTO)

## Контекст

Разработчики уже используют AI coding assistants (Claude Code, Cursor, OpenCode). MCP (Model Context Protocol) позволяет любому инструменту расширять возможности этих ассистентов через stdio. Complior может быть доступен не только через собственный TUI, но и как набор инструментов внутри существующих IDE.

## Решение

MCP Server как **Path C** — канал дистрибуции. Тот же TS engine, другой транспорт.

```
Внешний AI-инструмент (Claude Code / Cursor / OpenCode)
    ↕ MCP Protocol (stdio)
Complior MCP Server
    ↕ Прямые вызовы функций (тот же TS-процесс)
Complior TS Engine
```

### 8 MCP-инструментов

| Инструмент | Описание |
|------------|----------|
| `complior_scan` | Сканирование проекта на комплаенс |
| `complior_fix` | Исправление конкретной проблемы |
| `complior_score` | Текущий score комплаенса |
| `complior_explain` | Объяснение статьи регуляции |
| `complior_passport` | Agent Passport операции |
| `complior_validate` | Валидация комплаенса |
| `complior_deadline` | Дедлайны и сроки |
| `complior_suggest` | Рекомендации по улучшению |

### Нулевой дополнительный код

TUI использует HTTP/SSE к TS engine. MCP использует stdio к тому же engine. Код движка общий. MCP-сервер — ~200 строк маппинга stdio → вызовы функций движка.

## Обоснование

- **Дистрибуция**: разработчик пробует `complior_scan` в Claude Code → видит score → хочет полный дашборд → ставит TUI. Воронка: MCP → TUI → Pro.
- **Исследование**: данные использования MCP показывают, какие инструменты популярны → приоритизация P1 фич.
- **Ноль доп. работы**: тот же engine, ~200 строк для stdio-транспорта.
- **Бесплатная точка входа**: MCP всегда бесплатен, как часть Free-тарифа.

### Пример воронки

1. Разработчик добавляет `complior` в MCP-конфиг Claude Code
2. Claude автономно проверяет комплаенс при написании AI-кода
3. Видит: «Score: 42/100. 3 проблемы с Ст. 50.1»
4. Хочет: полный дашборд, split-diff, авто-фикс
5. Ставит: `npx complior` → полный TUI

## Последствия

- **Позитивные**: бесплатный канал дистрибуции, нулевые доп. затраты, данные использования, воронка конверсии.
- **Негативные**: MCP в чужом инструменте — ограниченный UX (нет дашборда, нет split-view).
- **Митигация**: ограниченный UX в MCP — это фича, не баг. Мотивирует переход на полный TUI.
