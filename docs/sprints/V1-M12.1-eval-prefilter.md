# V1-M12.1: Eval Pre-Filter

**Status:** 🔴 RED
**Created:** 2026-04-21
**Feature Area:** FA-02 (Eval Architecture) — see `docs/feature-areas/eval-architecture.md` §11
**Parent:** V1-M12 (Context-Aware Eval) — tech debt fix
**Goal:** Filter eval tests BEFORE execution (saves HTTP/LLM costs)

---

## Context

V1-M12 delivered profile filtering logic (`filterTestsByProfile`) and wired it into `eval-service.ts` — but the filter runs **AFTER** `runEval()` completes (lines 199-219). All tests still execute, costing the same HTTP/LLM calls. The filter only produces metadata (filterContext + disclaimer).

**This milestone moves filtering BEFORE execution** — the core cost-saving promise of V1-M12.

### What needs to change

In `eval-service.ts`, between lines 194 (profile loaded) and 196 (`runEval()` called):

1. Call `filterTestsByProfile()` on the test sources
2. Create a filtered `EvalTestSources` wrapper with only applicable tests
3. Pass the filtered sources to `runEval()` instead of unfiltered ones
4. Keep filterContext/disclaimer attachment (already done)

**~15-20 lines of code change.**
